## 2026-05-02 - SSRF and Prompt Injection via Unvalidated Issue URLs
**Vulnerability:** External input (`github_issue_url`) was directly interpolated into the user prompt passed to the LLM agent without validation in `agentic/workflows/jules_workflow.py` and `agentic/jules/agent.py`.
**Learning:** Passing unsanitized URLs to an LLM exposes the system to prompt injection (where a malicious URL contains instructions overriding the agent's intent) and SSRF (where the LLM agent could be tricked into querying internal or unintended domains).
**Prevention:** Always validate external URLs using strict parsing (like `urllib.parse`) before embedding them in prompts. Enforce allowed schemes (`https`), specific domains (`github.com`), and strictly validate path structures to ensure only intended inputs reach the LLM.
## 2026-05-24 - [CRITICAL] Prevent SSRF via MCP Server URL Initialization
**Vulnerability:** The `MCP_SERVER_URL` passed into `FastMCPToolset` in `agentic/jules/agent.py` and `agentic/workflows/jules_workflow.py` was not validated. An attacker could potentially supply a malicious URL (like cloud metadata endpoints, e.g., 169.254.169.254) causing a Server-Side Request Forgery (SSRF) when the agent makes network requests.
**Learning:** Initializing toolsets or any client that performs network operations with external/unvalidated URLs opens up the application to SSRF attacks, even if the URL is nominally intended for internal configuration. Cloud metadata endpoints are particularly sensitive.
**Prevention:** Always validate external or configuration-provided URLs used for external API connections. Enforce safe schemes (HTTP/HTTPS), block known metadata hostnames, and use DNS resolution (`socket.getaddrinfo`) coupled with IP checking against a blocklist to prevent bypasses like DNS rebinding or non-standard encodings.
