## 2026-05-02 - SSRF and Prompt Injection via Unvalidated Issue URLs
**Vulnerability:** External input (`github_issue_url`) was directly interpolated into the user prompt passed to the LLM agent without validation in `agentic/workflows/jules_workflow.py` and `agentic/jules/agent.py`.
**Learning:** Passing unsanitized URLs to an LLM exposes the system to prompt injection (where a malicious URL contains instructions overriding the agent's intent) and SSRF (where the LLM agent could be tricked into querying internal or unintended domains).
**Prevention:** Always validate external URLs using strict parsing (like `urllib.parse`) before embedding them in prompts. Enforce allowed schemes (`https`), specific domains (`github.com`), and strictly validate path structures to ensure only intended inputs reach the LLM.

## 2026-05-23 - SSRF Vulnerability via Unvalidated MCP Server URLs
**Vulnerability:** The `MCP_SERVER_URL` was taken from the environment and directly passed to `FastMCPToolset` in `agentic/workflows/jules_workflow.py` and `agentic/jules/agent.py` without validation. This allowed an attacker to point the MCP toolset to an arbitrary internal IP or cloud metadata service (e.g., AWS/GCP metadata), exposing the system to Server-Side Request Forgery (SSRF) since the toolset performs network requests to discover tools.
**Learning:** Initializing remote toolsets/servers with external URLs or user-configurable environment variables can lead to SSRF. Simply checking the URL string is insufficient because attackers can use DNS rebinding or obfuscated IPs (like octal/decimal encoding) to bypass basic hostname string matches.
**Prevention:** Always validate external URLs thoroughly before establishing connections. Validation must include: strict scheme checking (e.g., http/https), stripping IPv6 brackets, resolving the hostname to its underlying IPs using `socket.getaddrinfo`, and using an IP address evaluation library (like `ipaddress`) to explicitly block internal, private, and cloud metadata subnets/IPs (e.g., 169.254.169.254, fd00:ec2::254).
