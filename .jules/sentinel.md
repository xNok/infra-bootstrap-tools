## 2026-05-02 - SSRF and Prompt Injection via Unvalidated Issue URLs
**Vulnerability:** External input (`github_issue_url`) was directly interpolated into the user prompt passed to the LLM agent without validation in `agentic/workflows/jules_workflow.py` and `agentic/jules/agent.py`.
**Learning:** Passing unsanitized URLs to an LLM exposes the system to prompt injection (where a malicious URL contains instructions overriding the agent's intent) and SSRF (where the LLM agent could be tricked into querying internal or unintended domains).
**Prevention:** Always validate external URLs using strict parsing (like `urllib.parse`) before embedding them in prompts. Enforce allowed schemes (`https`), specific domains (`github.com`), and strictly validate path structures to ensure only intended inputs reach the LLM.
## 2025-02-23 - SSRF Vulnerability in MCP Server URL Initialization
**Vulnerability:** The Jules agent and Prefect workflows instantiate `FastMCPToolset` with an `MCP_SERVER_URL` parameter that lacked proper URL validation. This allowed Server-Side Request Forgery (SSRF) attacks, wherein the agent could be tricked into querying internal network resources (such as `169.254.169.254` or `metadata.google.internal`) instead of a legitimate MCP server.
**Learning:** External user inputs and configurable environment variables that drive network connections must be strictly validated. Pydantic AI's `FastMCPToolset` connects to the URL provided during instantiation without natively sanitizing or checking for SSRF to local/cloud-metadata endpoints.
**Prevention:** A specific validation function (`is_safe_mcp_server_url`) was introduced in `agentic/jules/url_validation.py`. It explicitly enforces valid URL schemas (`http`, `https`) and blocks known cloud metadata addresses and link-local ranges by matching on hostnames and testing valid IP equivalents (stripping brackets for IPv6 compatibility). All initialization points for `FastMCPToolset` must route through this validation first.
